Relational database

Relational databases are a type of database that stores and provides access to data points that are related to one another, or hold similar type of data. Relational databases hold data in tables, with rows holding a collection of similar, or related values, and columns holding data of a particular type, or attribute. Relational databases make use of keys to uniquely identify each row.


SQL

Structured Query Language (SQL) is the standard programming language used to manipulate data in relational databases. It is used in databases such as MySQL, SQL Server, MS Access, Oracle, Sybase, Informix, etc.


SQL Vs MySQL

SQL is the programming language that is used to manipulate data in relational databases, while MySQL is the relational database management system that makes use of SQL. Below are some differences sourced from here

SQL MySQL
SQL is a query programming language that manages RDBMS MySQL is a relational database management system that uses SQL
SQL is primarily used to query and operate database systems MySQL allows you to handle, store, modify and delete data and store data in an organized way
SQL does not support any connector MySQL comes with an in-built tool known as MySQL Workbench that facilitates creating, designing, and building databases
SQL follows a simple standard format without many or regular updates MySQL has numerous variants and gets frequent updates
SQL supports only a single storage engine MySQL offers support for multiple storage engines along with plug-in storage, making it more flexible
SQL does not allow other processors or even its own binaries to manipulate data during execution MySQL is less secure than SQL, as it allows third-party processors to manipulate data files during execution

Database Normalization

Database Normalization is a design technique the ensures, there databases are organized to avoid data redundancy and database anomalies (insertion anomaly, update anomaly, and deletion anomaly)
Database Normalization comes in 6 forms:

  • 1st Normal Form
  • 2nd Normal Form
  • 3rd Normal Form
  • Boyce-Codd Normal Form (BCNF)
  • 4th Normal Form
  • Fifth Normal Form
Database Normalization often involves split a database into multiple tables, with defined relations achieve the above mentioned results


Creating a database

The code snippet is used to create a database

CREATE DATABASE ^database_name

This is used to delete a database

DROP DATABASE ^database_name

^database_name refers to the name of your database to be created or deleted (pretty self-explanatory)


Creating a table

Within a database are many tables that hold specific data The code snippet is used to create a table

CREATE TABLE ^table_name

This is used to delete a table

DROP TABLE ^table_name

^table_name refers to the name of the table to be created or deleted


Select item from table

There are many ways to select an item from a table, depending on how the data is organized, and the type of data to be selected. Below are a few examples:

  • Using SELECT

    Select items from a specified column(s) in a table

    SELECT ^column1, column2,... FROM ^table_name

  • Select all the items in the specified table

    SELECT * FROM ^table_name

The SELECT keyword is used with other conditionals to allow for more precision in data retrieval. Find how this is used at w3schools
